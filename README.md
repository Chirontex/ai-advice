# ai-advice

Мультиагентная ИИ-система персонального консультирования на базе [Kilo Code](https://kilocode.ai). Помогает решать широкий спектр жизненных и профессиональных задач — от бытовых вопросов до юридических, медицинских, финансовых, технических, творческих и организационных.

## Архитектура

Система построена по принципу «оркестратор + специализированные субагенты»:

```
Советник (advice.xml)
  ├── Лекс (law.md)       — юридический агент
  ├── Медик (medical.md)  — медицинский агент
  └── Ридли (produce.md)  — продюсерский агент
```

**Советник** (`advice.xml`) — универсальный оркестратор. Принимает запрос пользователя, классифицирует его по доменам, при необходимости привлекает специализированных субагентов через механизм `task`, синтезирует информацию из нескольких источников и выдаёт структурированный ответ с конкретным планом действий.

## Субагенты

| Агент | Файл | Домен | Примеры задач |
|-------|------|-------|---------------|
| **Лекс** | `law.md` | Право | Анализ договоров, составление исков, правовые консультации, комплаенс, сравнительно-правовой анализ. Приоритетная экспертиза — законодательство РФ |
| **Медик** | `medical.md` | Медицина | Дифференциальная диагностика, рекомендации по лечению и обследованию, проверка доступности препаратов на рынке РФ, экстренные состояния |
| **Ридли** | `produce.md` | Продакшн | Кинопроизводство, музыка, подкасты, ивенты, цифровой контент. Бюджетирование, календарное планирование, кастинг, дистрибуция, питчинг |

Субагенты работают автономно: получают полный контекст задачи, проводят собственное исследование (включая веб-поиск через инструменты оркестратора — Tavily/webfetch), и возвращают готовый результат оркестратору для финального синтеза. Каждый субагент имеет чёткие границы компетенции и при необходимости рекомендует обратиться к смежному агенту.

## Возможности Советника

- **Автоматическая классификация** запроса по домену (право, медицина, финансы, техника, кулинария, быт, путешествия)
- **Параллельная обработка** — при смешанных вопросах запускает несколько субагентов одновременно (например: `law + medical` при ДТП)
- **Веб-исследование** — многоуровневый поиск через Tavily MCP (от быстрого извлечения до глубокого исследования и обхода сайтов) с автоматическим fallback'ом на прямую загрузку страниц при недоступности Tavily
- **Обход блокировок** — при недоступности страницы через Tavily или прямую загрузку использует Playwright (JavaScript-рендеринг, обход Cloudflare)
- **Точные расчёты** — все финансовые, налоговые и прочие вычисления выполняются через Python
- **Адаптивный формат ответа** — от 2–4 предложений для простых вопросов до многостраничного структурированного разбора для сложных межотраслевых ситуаций
- **Структурированное рассуждение** — Sequential Thinking MCP позволяет Советнику вести пошаговый анализ с проверкой гипотез, возвратом к предыдущим шагам и ветвлением логики (особенно полезно для юридических, медицинских и финансовых кейсов)

## MCP-серверы

Советник использует два MCP-сервера, расширяющих его исследовательские и аналитические возможности. Подключение настраивается в конфигурации Kilo Code (`~/.config/kilo/kilo.jsonc`, секция `mcp`).

### Tavily Search

Обеспечивает веб-поиск, извлечение контента, обход сайтов и глубокое исследование тем.

**Получение API-ключа:**
1. Зарегистрируйтесь на [tavily.com](https://tavily.com)
2. Создайте API-ключ в [личном кабинете](https://app.tavily.com/home)
3. Бесплатный тариф — 1000 запросов/месяц

**Вариант А: локальный сервер (через npx)**

```jsonc
"tavily-search": {
  "type": "local",
  "command": ["npx", "-y", "tavily-mcp@latest"],
  "env": {
    "TAVILY_API_KEY": "<ваш-ключ>"
  },
  "enabled": true
}
```

**Вариант Б: удалённый сервер Tavily (официальный облачный MCP)**

Tavily предоставляет готовый облачный MCP-эндпоинт — не нужно ничего разворачивать самостоятельно:

```jsonc
"tavily-search": {
  "type": "remote",
  "url": "https://mcp.tavily.com/mcp/?tavilyApiKey={api-key}",
  "enabled": true
},
```

Преимущество удалённого варианта: не требует Node.js/npx на локальной машине, обновления сервера происходят на стороне Tavily автоматически.

**Доступные инструменты:**
| Инструмент | Назначение |
|---|---|
| `tavily_search` | Веб-поиск по запросу (сниппеты + ссылки) |
| `tavily_extract` | Извлечение содержимого с конкретных URL (basic/advanced) |
| `tavily_research` | Глубокое исследование темы с синтезом из множества источников |
| `tavily_crawl` | Обход страниц сайта на заданную глубину |
| `tavily_map` | Построение карты URL сайта |

> **Примечание:** Советник спроектирован с расчётом на возможную недоступность Tavily (отключён в конфиге, исчерпаны кредиты, сбой сервиса). При недоступности он автоматически переключается на fallback-стратегию: `webfetch` + собственные знания об авторитетных источниках. Система остаётся работоспособной и без Tavily — просто с меньшей глубиной поиска.

### Sequential Thinking

Обеспечивает структурированное многошаговое рассуждение с возможностью пересмотра предыдущих шагов, ветвления логики и проверки гипотез.

**Установка (WSL2 на Windows):**

Node.js устанавливается внутрь WSL2 (Ubuntu), на хост-систему — ничего:

```bash
# В WSL2:
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**Конфигурация Kilo (`kilo.jsonc`):**

```jsonc
"sequential-thinking": {
  "type": "local",
  "command": ["wsl", "bash", "-c", "npx -y @modelcontextprotocol/server-sequential-thinking"],
  "enabled": true
}
```

На нативной Linux-машине (без WSL2) команда проще:

```jsonc
"sequential-thinking": {
  "type": "local",
  "command": ["npx", "-y", "@modelcontextprotocol/server-sequential-thinking"],
  "enabled": true
}
```

**Диагностика при проблемах:**

| Проблема | Проверить |
|---|---|
| Kilo показывает ошибку подключения | `wsl bash -c "npx -y @modelcontextprotocol/server-sequential-thinking"` в PowerShell — запускается ли вручную? |
| `npx` не может скачать пакет | Доступ к npm registry: `curl -s https://registry.npmjs.org/` в WSL2 |
| Node.js не установлен в WSL2 | `wsl bash -c "node --version"` |
| WSL2 не установлен | `wsl --status` в PowerShell. Если нет — [инструкция по установке](https://learn.microsoft.com/ru-ru/windows/wsl/install) |

После изменения конфигурации Kilo требуется перезапуск (`/exit` и открыть заново). MCP-серверы подхватываются автоматически — `npx` скачает пакеты при первом запуске и закеширует.

## Установка и использование

1. Установите [Kilo Code](https://kilocode.ai) в VS Code
2. Клонируйте репозиторий:
   ```bash
   git clone git@github.com:Chirontex/ai-advice.git
   ```
3. Настройте `advice.xml` как системный промпт агента Kilo Code (см. [документацию Kilo](https://kilocode.ai/docs))
4. Субагенты (`law.md`, `medical.md`, `produce.md`) подключаются как `subagent_type` в конфигурации агента `task`

## Сторонние источники данных

Субагенты используют следующие внешние источники для получения актуальных данных (помимо веб-поиска через MCP-серверы Советника):

- **Юриспруденция:** GitHub-репозитории с MCP-серверами российского права, правовые базы
- **Медицина:** Клинические рекомендации МЗ РФ, PubMed, Cochrane, Vidal.ru, RLS.ru, ГРЛС (Государственный реестр лекарственных средств)
- **Продакшн:** Фестивальные платформы, грантовые порталы (Президентский фонд культурных инициатив, Фонд кино и др.), стриминговые сервисы
